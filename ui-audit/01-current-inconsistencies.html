Layout & Container Inconsistencies

The app uses conflicting max-width values and padding across different pages, creating an inconsistent visual experience as users navigate between sections.

Max-Width Variations

Page Max-Width Actual Pixels
TopNavigationLayout max-w-[1440px] 1440px
ProjectsPage max-w-screen-lg ~1024px
AccountSettingsPage max-w-6xl ~1152px
Products/Shots No limit ~1408px usable

Visual Impact

Projects page appears narrower than other pages, creating jarring transitions when navigating. Content doesn't fill available space within the global container.

Padding Misalignment

Location Desktop Mobile
Top Nav Bar px-6 (24px) px-4
Main Content px-8 (32px) px-4
PageHeader content-padding 24px

Visual Impact

Header items and main content don't align vertically. The 8px difference between nav (24px) and content (32px) creates a visible "step" in the layout.

Badge & Status Inconsistencies

The app has three badge systems (Badge, StatusBadge, TagBadge) but components often create inline badge styles instead of using these components.

Current Badge Variations

Same semantic meaning, different implementations:

Active/Complete Status
Active (StatusBadge) Complete (inline) Success (Badge)
Pending/Warning Status
Pending (StatusBadge) In Progress (inline) Warning (Badge)
Error/Archived Status
Failed (StatusBadge) Archived (inline) Error (Badge)

10+ instances of inline badge styling

Components define badge colors inline instead of using StatusBadge or TagBadge components. This leads to inconsistent colors, radii, and padding.

Shape Inconsistencies

Component Shape Expected

TagBadge rounded-full Correct

Form Input Inconsistencies

Form inputs show variations in focus ring width, border radius, and dark mode handling across different parts of the application.

Focus Ring Variations

Location Ring Width Ring Opacity Dark Mode
input.jsx (design system) ring-2 100% Yes
ShotProductAddModal.jsx ring-1 100% Yes
BulkTaggingToolbar.jsx ring-2 20% Partial
ExpandableSearch.jsx ring-2 60% Yes

Accessibility Concern

Inconsistent focus indicators make it harder for keyboard users to track focus. The ring-2 ring-primary/20 pattern is too subtle for WCAG compliance.
