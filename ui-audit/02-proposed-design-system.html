7. Design Rules

Guidelines for maintaining consistency as the application grows.

DO: Use Design Tokens

Always use semantic tokens instead of arbitrary values. This ensures consistency and makes future updates easier.

Do
className="rounded-card"
className="gap-3"
className="px-6"
Don't
className="rounded-lg"
className="gap-[13px]"
className="px-[26px]"

DO: Use Semantic Status Components

Use StatusBadge for status indicators and TagBadge for user tags. Never create inline badge styles.

Do
<StatusBadge status="active">
<TagBadge color="blue">
Don't
<span className="bg-emerald-100 text-emerald-700 px-2 py-0.5 rounded-full">

DO: Consistent Focus States

All interactive elements must have a visible focus indicator with ring-2 for accessibility compliance.

Do
focus:ring-2 focus:ring-primary
Don't
focus:ring-1 focus:ring-primary/20

DO: Align Sticky Elements

Use consistent sticky positioning with proper z-index layering to prevent visual conflicts.

Stack Order
TopNav: z-50, top-0
PageHeader: z-40, top-14
BulkToolbar: z-30, top-28
