Design Rationale

Style numbers belong to the family, not the colorway

A product family has one dominant style number (e.g. M-BT-PN-1252) and optionally 1–3 aliases from previous seasons or alternate naming. This is family-level metadata, not per-SKU. Moving it to the header eliminates the most repeated text on the page.

Compressed size ranges reduce noise

Instead of enumerating every composite size (S/30, S/32, M/30, M/32, L/30, L/32...), the family header shows the range and dimensions separately: S – XXL / Inseams: 30", 32". Colorway cards only show a size note when their range differs from the family default.

Colorway cards are scannable

Each card shows: thumbnail, color swatch, color name. Status badges (Discontinued, New) appear inline. A size override line only appears when needed. No SKU codes, no repeated family metadata. The result is a grid that can be scanned in under 2 seconds.

Mobile is a simplified vertical stack

Full-width product image, family identity, and compact colorway list. Same information hierarchy, adapted for thumb-scrolling. On mobile, colorway cards are single-column with smaller thumbnails.
