Error States

Standardized error handling: ErrorBoundary for crashes, banners for network issues, centered pages for 403/404, inline for form validation, Sonner toasts for transient feedback.

ErrorBoundary: Chunk Load Error

Shown when a lazy-loaded route chunk fails (deploy, network). Two actions: "Try again" resets the boundary, "Reload app" forces a full page reload. Matches existing ErrorBoundary.tsx.

Reload required
A required part of the app failed to load. This can happen after an update or on an unstable connection. Reload to continue.
Build abc123
ErrorBoundary: Generic Error

Catch-all for unexpected runtime errors. Single "Try again" button resets the boundary. Error message shown in muted text.

Something went wrong
Cannot read properties of undefined (reading 'shots')
Build abc123
Network Banners (Top of Page)

Persistent banners for connectivity issues. Rendered at the top of AppShell, above the page content. Amber for offline (recoverable), red for server errors.

Offline Banner (Amber)
You are offline. Changes will sync when reconnected.
Shown when navigator.onLine === false. Dismissed automatically on reconnect.
Network Error Banner (Red)
Unable to connect to server. Please try again later.
Shown on Firestore connection failure. Manual "Retry" re-attempts the subscription.
