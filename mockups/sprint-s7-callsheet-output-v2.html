Design Decisions — v2 vs v1

Key changes based on user feedback and SetHero competitor research.

1. Modular Grid Header (SetHero-inspired)
The 3-zone header compresses company info, production details, and date/weather into approximately 120px of vertical space. The previous v1 design used more than half the first page with stacked key-value pairs before reaching the schedule. The hero crew call time (42px) is the single most scanned element on any call sheet — it now dominates the center zone. Hospital info is in the left zone at a glance for safety compliance.
2. Multi-Track Schedule Timeline
The schedule now renders overlapping events side-by-side in a 2-column split rather than stacking sequentially. This mirrors how the schedule builder works in the app and gives crew a realistic view of concurrency. Time-aligned layout with color-coded block types (Setup = amber, Shooting = blue, HMU = purple, Meal = green, Travel = gray) is scannable at a glance. Each shooting block includes shot numbers, setup count, product categories, and lighting notes.
3. Department-Grouped Crew Grid (SetHero pattern)
Crew are grouped by department with dark header bands containing walkie channel chips. Each department is a self-contained block with break-inside: avoid for clean print pagination. The 2-column toggle allows dense side-by-side packing for productions with 40+ crew. Phone numbers are included as a 4th column — this was a gap in v1. Zebra striping uses the neutral-50 token for subtle differentiation.
4. Print-First Document Design
Every element is sized for US Letter (816px wide). Print styles use @page { size: letter; margin: 0.5in 0.4in }, force background color printing, hide screen controls, and enforce orphan control on department blocks. Page 2 gets a running header with production name, day count, and date for context when pages separate. Footer includes confidentiality notice and page numbers on both pages.
5. Section Visibility Toggles
The toggle bar provides granular control over Header, Cast, Schedule, Crew, and Notes sections. A producer sending to crew includes everything. A client-facing version hides crew details and phone numbers. The "Print Preview" mode removes shadows and shows exact page boundaries. All toggles work with smooth CSS transitions. This is a direct response to user feedback requesting field visibility control.
6. Mobile Responsive
Below 860px, the header grid stacks vertically, the timeline drops to single-column, and the crew grid forces 1-column. The document fills the viewport width without a centered card. This allows crew to read the call sheet on their phone without pinch-zooming. The controls bar wraps gracefully and remains sticky at the top.
7. Fashion/Commercial Context
The design uses the "Fashion-Forward" (Option C) base from v1 — near-black ink (#111111), minimal accent color (red brand moments on the top/bottom rules only), editorial typography (Inter 300-700). Shot count summary and product counts on page 1 are specific to fashion/commercial production. The advance preview for the next shoot day is a detail borrowed from SetHero that production coordinators find invaluable for overnight prep.
8. Cast Grid Enhancements
Added a "Report" column between Pickup and HMU for the time talent checks in at basecamp. Status badges (S/W/H) are now colored chips instead of plain text for faster scanning. The legend row is explicit and includes a note directing readers to production notes for talent-specific instructions. Column widths are proportional to content importance — name and role get the most space, times are right-aligned for vertical scanning.
Implementation Notes
  • All design tokens match the project specification exactly — no arbitrary hex values or font sizes
  • Table header: bg=#000000, text=#ffffff, font-size=9px, bold, uppercase, 0.05em tracking
  • Table body: font-size=11px, row-height=20px
  • Section bands: bg=#111111, text=#ffffff, 11px bold, 0.08em tracking, uppercase
  • Department headers use the same spec as section bands for visual consistency
  • Accent color (#E31E24) used only for top/bottom rules and hospital label — brand moments only
  • Responsive breakpoint at 860px covers the document width + padding threshold
  • Print styles tested with Chrome "Save as PDF" and "Print" dialog
  • Status badges use established color tokens: blue for Start, amber for Work, gray for Hold
  • The walkie chip in department headers uses rgba white for subtle contrast against the dark band
