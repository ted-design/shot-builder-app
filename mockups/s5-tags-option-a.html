Design Notes

  • Category-based coloring — 4 fixed palettes. Simple to implement, no per-tag config needed.
  • Sort order: priority → gender → media → other. Within each category, alphabetical.
  • Size: text-xxs (11px), rounded-md, px-2 py-0.5. Compact and information-dense.
  • Dark mode: Uses rgba backgrounds at 12% opacity for consistent, non-garish tints.
  • Implementation: Map ShotTagCategory to a CSS class. No changes to Firestore data model.
  • Pros: Instant visual parsing of tag type. Predictable — same category always same color.
  • Cons: Less granular — "High Priority" and "Low Priority" are same warm tone. Custom tags all gray.
