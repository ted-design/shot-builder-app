Design Notes (Phase 2c — Mobile & Tablet)

  • Item count (mobile drawer, project context): 7 items (Back, Shots, Pulls, Assets, Products, Library, Account). Down from 12+ in current mobile sidebar. Desktop-only items (Call Sheet, Admin, Settings) are hidden.
  • Item count (mobile drawer, org context): 4 items (Dashboard, Products, Library, Account). Clean and minimal.
  • Touch targets: All nav items are 44px minimum height (mobile-nav-item class).
  • Surface badges: Optional "Limited" / "Reader" badges shown in tablet drawer to hint at mobile capability level. These are informational, not functional — they help new users understand what they can do on their device.
  • Desktop-only redirect: If a user navigates to a Call Sheet / Admin / Settings URL on mobile (e.g. from a shared link), they get redirected to the project dashboard with a Sonner toast: "Call sheets are available on desktop."
  • Project switcher on phone: Top bar shows project name with a chevron. Tapping opens a project switch sheet (bottom sheet pattern). On org-level pages, top bar just shows "All Projects" with no chevron.
  • Tablet drawer (768px): Wider at 320px (vs 288px on phone). Includes a swipe-hint bar on the right edge. Swipe left to close. Top bar gains a breadcrumb and search icon since there's more horizontal space.
  • Library on mobile: Tapping "Library" in mobile drawer goes directly to a tabbed Library page (Talent / Crew / Locations / Palette tabs) rather than expanding a sub-menu. Simpler for one-handed use.
  • Backdrop: Dark 50% overlay closes drawer on tap. Same pattern as current MobileSidebar.
  • No horizontal scroll: All content fits within viewport width at both 375px and 768px.
