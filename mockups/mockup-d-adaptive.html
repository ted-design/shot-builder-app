How Approach D works

Information-Weighted Blocks

Time periods with more overlapping events across streams get more vertical space. A busy 70-minute block takes more room than a sparse 2-hour gap.

Relative Accuracy Within Blocks

Within each dense block, card Y-position and height proportionally reflect start times and durations relative to one another.

Compressed Dead Time

Gaps and sparse periods (load-in, lunch) shrink to a minimal indicator instead of wasting screen real estate on empty space.

Readable Card IA

Cards from Approach A: semantic icon colors, full metadata rows, tag badges. Content naturally fits because the block scales to accommodate it.

Cross-Track Alignment

Photo and Video events starting at the same time appear at the same vertical position. Overlapping durations are visually clear across tracks.

Adaptive px/min Rate

Each block computes its own scale. Dense Block 1 uses ~8px/min. Moderate Block 2 uses ~5px/min. Banners use a fixed height.
